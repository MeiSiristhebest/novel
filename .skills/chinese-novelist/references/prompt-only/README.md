# Prompt-Only 控制面（非脚本版本）

本目录提供一套不依赖 Python/PowerShell 的手工写作闭环协议。

## 已有能力
- 战略层：主赌注、主编审稿、删改权、反馈分流、声纹审计
- 发布性层：排版、细节、声纹、同质化、综合发布性
- 高阶层：危险下注、审美升级、动态读者行为学、人物压场、记忆点实验室

## 核心协议
1. `MASTER_ORCHESTRATOR.md`：总控入口
2. `PROGRESSIVE_DISCLOSURE_PROTOCOL.md`：渐进式披露与分层读取协议
3. `FILE_RELATION_MAP.md`：文件之间的依赖关系图
4. `GATE_PROTOCOL.md`：门禁统一协议
5. `CONTEXT_PACKER_PROMPT.md`：按层打包，不持久化全文注入
6. `OUTPUT_CONTRACTS.md`：统一输出与包文件契约

## 核心原则
1. `.context` 是唯一事实源（SSOT）。
2. 实时信息只能校准，不得主导创作。
3. 高阶角色负责接近“顶级作者能力”，不是门禁替代品。
4. 任何角色都必须按 `PROGRESSIVE_DISCLOSURE_PROTOCOL.md` 分层读取，不得一次性全量展开。
5. 每个高阶文件都必须能在 `FILE_RELATION_MAP.md` 中找到自己的上游、下游和回写去向。
6. 每个关键 gate 都必须能说明自己的证据来源、失败去向和阻断策略。
