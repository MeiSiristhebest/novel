# SOP_MANUAL_RUN（手工执行）

## 一、准备
1. 打开 `MASTER_ORCHESTRATOR.md`。
2. 先看 `PROGRESSIVE_DISCLOSURE_PROTOCOL.md` 和 `FILE_RELATION_MAP.md`。
3. 从项目按 L0 -> L1 -> L2 -> L3 的顺序复制文件到会话，不得一次性全贴。

## 二、信息沉淀流程
1. 有新信号先跑 `signal_curator`。
2. 先执行 `LIVE_SIGNAL_DECAY_RULE`，再决定是否采纳。
3. 采用的信号写入 `LIVE_SIGNAL_INGESTION.md`。
4. 考古结论写入 `ARCHAEOLOGY_LOG.md`。
5. 未通过 `SIGNAL_VALUE_GATE` 的信号不进入正文流程。

## 三、高阶强化流程
1. 新卷/新 Arc/长线滞涩时，先跑 `dangerous_bet_lab`。
2. 每 5 章至少跑一次 `aesthetic_upgrade_director`。
3. 出现读者疲态时，跑 `reader_behavior_modeler` 而不只看静态画像。
4. 关键章节在 `writer` 后、`qa` 前跑 `miracle_moment_lab`。
5. `ruthless_cut_review` 输出中必须维护 `SACRIFICE_LEDGER.md`。

## 四、引用标准
1. 每个本轮使用的文件都要写明“为什么读它”。
2. 每个高阶角色输出都要写 `reference_files`，且这些文件必须能在 `FILE_RELATION_MAP.md` 里找到上下游关系。
3. 如果某文件没有上游、下游、回写去向，默认视为设计不完整，不应进入长期体系。
