# 专家模式检查清单 (Expert Mode Checklist)

> **触发词**：`专家模式` / `创建项目：[书名]` / `空项目`
> **核心行为**：无访谈 → 一次性生成 8 个空模板文件

---

## 📋 执行清单

AI 收到专家模式请求后，**必须**执行以下步骤：

### Step 1: 获取项目名称

- [ ] 从用户输入中提取书名
  - 如果用户说"创建项目：[书名]"，使用 [书名]
  - 如果用户只说"专家模式"，询问："请输入小说名称："

### Step 2: 创建项目目录

- [ ] 创建 `novels/[书名]/` 目录

### Step 3: 批量生成空模板

- [ ] 从 `references/templates/` 读取模板
- [ ] 生成以下 8 个文件（仅标题和占位符）：

| 文件 | 内容 |
|:-----|:-----|
| `00-策划书.md` | `# [书名] 策划书\n\n（待填写）` |
| `01-世界观圣经.md` | `# [书名] 世界观圣经\n\n（待填写）` |
| `02-人物深度档案.md` | `# [书名] 人物档案\n\n（待填写）` |
| `03-分章细纲.md` | `# [书名] 分章细纲\n\n（待填写）` |
| `04-伏笔追踪表.md` | `# [书名] 伏笔追踪表\n\n（待填写）` |
| `05-时间线.md` | `# [书名] 时间线\n\n（待填写）` |
| `06-细节追踪表.md` | `# [书名] 细节追踪表\n\n（待填写）` |
| `07-项目状态.md` | `# [书名] 项目状态\n\n阶段：策划中\n当前章：0` |

> 可选：`08-素材碎片.md`（灵感池/金句库；推荐用 08，避免与 06-细节追踪表 冲突）。

### Step 4: 输出确认

- [ ] 输出以下格式的确认信息：

```
✅ 专家模式：项目框架已创建。

📁 位置：novels/[书名]/
📄 文件：
  - 00-策划书.md
  - 01-世界观圣经.md
  - 02-人物深度档案.md
  - 03-分章细纲.md
  - 04-伏笔追踪表.md
  - 05-时间线.md
  - 06-细节追踪表.md
  - 07-项目状态.md

所有文件均为空模板，请自行填写。
🚀 准备好后，告诉我"开始写第一章"。
```

---

## ⚠️ 禁止行为

1.  **禁止访谈**：专家模式不进行任何问答。
2.  **禁止填充内容**：文件只包含标题和占位符，不自动生成任何实际内容。
3.  **禁止渐进确认**：一次性生成所有 8 个文件，无需逐个确认。
