# 修订工作流 (Revision Workflow)

> **核心理念**：好文章是改出来的，不是写出来的。修订是专业作家的核心能力。

---

## 📊 修订的重要性

### 为什么要修订？

```
初稿 vs 终稿的区别：

初稿问题：
- 逻辑不通顺
- 描写不够生动
- 对话不够自然
- 节奏不够紧凑
- 错别字和病句

修订后：
- 逻辑清晰
- 描写精准
- 对话鲜活
- 节奏流畅
- 文字干净
```

### 专业作家的时间分配

| 阶段 | 时间占比 | 说明 |
|------|----------|------|
| 构思 | 10% | 想清楚要写什么 |
| 初稿 | 40% | 快速完成初稿 |
| 修订 | 40% | 精修打磨 |
| 校对 | 10% | 最后检查 |

## 🔄 三轮修订法

### 第一轮：结构修订（宏观）

**目标**：确保故事骨架正确

```markdown
检查清单：

□ 章节目标是否明确？
□ 主线是否有推进？
□ 节奏是否合适？
□ 伏笔是否铺设/回收？
□ 高潮是否足够？
□ 结尾钩子是否有力？

常见问题：
- 章节太长 → 拆分
- 章节太水 → 删减或合并
- 缺少冲突 → 增加障碍
- 高潮太弱 → 加强对抗
```

### 第二轮：场景修订（中观）

**目标**：确保每个场景都精彩

```markdown
检查清单：

□ 场景目的是否明确？
□ 冲突是否存在？
□ 感官描写是否到位？
□ 对话是否自然？
□ 情绪是否传达？
□ 过渡是否流畅？

常见问题：
- 描写太少 → 增加五感
- 描写太多 → 精简留核心
- 对话干瘪 → 增加潜台词
- 节奏拖沓 → 删减过渡
```

### 第三轮：语言修订（微观）

**目标**：确保每句话都精准

```markdown
检查清单：

□ 有无错别字？
□ 有无病句？
□ 有无重复用词？
□ 有无AI味表达？
□ 句子是否通顺？
□ 用词是否准确？

常见问题：
- 错别字 → 使用校对工具
- 重复词 → 同义词替换
- AI味 → 参考ai-guardrails.md
- 长句 → 适当断句
```

## ⏱️ 修订时机

### 日更模式下的修订

```
方案一：写完即改
写完一章 → 快速浏览修订 → 发布
优点：及时
缺点：可能改不彻底

方案二：隔日修订
今天写 → 明天改昨天的 → 发布
优点：冷静审视
缺点：需要存稿

方案三：周末集中改
周一到周五写 → 周末统一修订
优点：效率高
缺点：前面章节已发布

推荐：方案二（隔日修订）
```

### 存稿模式下的修订

```
最佳流程：

第1天：写初稿
第2天：第一轮结构修订
第3天：第二轮场景修订
第4天：第三轮语言修订
第5天：最终校对
第6天：发布

这样每章都经过5天的沉淀和打磨！
```

## 🔧 修订技巧

### 1. 冷却法

```
写完后放置24-48小时再改

原因：
- 刚写完时"当局者迷"
- 过一段时间能以读者视角审视
- 更容易发现问题

技巧：
- 写完先写下一章
- 回头改前一章
- 形成流水线
```

### 2. 朗读法

```
大声朗读你写的内容

能发现：
- 不通顺的句子
- 对话不自然
- 节奏问题
- 重复啰嗦

读不顺的地方就是需要改的地方！
```

### 3. 删减法

```
尝试删掉10-20%的字数

删减对象：
- "的"字过多
- 无意义的过渡
- 重复的信息
- 冗余的描写

原则：如果删掉不影响理解，就删掉
```

### 4. 替换法

```
找出频繁使用的词，替换掉

常见重复词：
- 然后、于是、接着
- 很、非常、十分
- 看着、望着、盯着
- 说道、问道、答道

解决：
1. Ctrl+F 搜索
2. 统计出现次数
3. 同义词替换
4. 句式重构
```

### 5. 对比法

```
对比自己心中的标杆作品

步骤：
1. 找到类似场景
2. 对比标杆作品的写法
3. 分析差距
4. 学习改进
```

## 🔧 进阶修订技巧 (Advanced Techniques)

### 6. 杀死你的宠儿 (Kill Your Darlings)

> 威廉·福克纳：“写作时，一定要杀死你的宠儿。”

*   **定义**：删掉那些你特别喜欢、写得特别漂亮，但对故事没有任何推动作用的段落/人物/场景。
*   **判断标准**：
    *   *这一段真的很美，但如果删掉它，故事还能继续吗？* -> 能 -> **删！**
    *   *这个配角很有趣，但他除了搞笑还有别的作用吗？* -> 没有 -> **删！**
*   **心态**：不要觉得浪费。它们是你练笔的过程，不是必须保留的结果。

### 7. Beta 读者反馈 (Beta Reader Feedback)

*   **找谁读？**：不要找会夸你的人（比如妈妈）。找爱看同类小说的读者。
*   **问什么？**：
    *   *❌ 这一章好不好看？*（太笼统）
    *   *✅ 你在哪里感到无聊了？*
    *   *✅ 你最讨厌哪个角色？*
    *   *✅ 你有没有猜到结局？*
*   **怎么听？**：
    *   *原则*：如果一个人说这里有问题，他可能是错的。如果五个人都说这里有问题，那一定是真的有问题。
    *   *原则*：读者只能指出问题（痛点），但他们给出的修改建议通常是不靠谱的。你是医生，你要自己开药。

## 📋 修订检查清单

### 结构层面

```markdown
□ 本章在全书中的作用清晰吗？
□ 本章推进了主线吗？
□ 本章有冲突吗？
□ 本章有爽点/情绪点吗？
□ 结尾有钩子吗？
□ 与前后章衔接自然吗？
```

### 场景层面

```markdown
□ 每个场景有目的吗？
□ 场景的视觉画面清晰吗？
□ 人物的动作和表情有吗？
□ 对话有潜台词吗？
□ 情绪传达到位吗？
□ 场景转换流畅吗？
```

### 语言层面

```markdown
□ 错别字消除了吗？
□ 病句修正了吗？
□ 重复词替换了吗？
□ AI味语句修正了吗？
□ 句子长度适中吗？
□ 节奏感好吗？
```

## 🚫 常见修订误区

### 误区一：过度修订

```
症状：一章改十遍还不满意
原因：完美主义
解决：设定修订次数上限（最多3遍）
```

### 误区二：边写边改

```
症状：写一段改一段，效率极低
原因：害怕写得不好
解决：先完成初稿，再统一修订
```

### 误区三：只改字词

```
症状：只改错别字，不改结构
原因：不知道怎么改
解决：学习三轮修订法，从结构开始
```

### 误区四：不改

```
症状：写完就发，从不修订
原因：时间不够/不重视
解决：认识到修订的价值，调整时间分配
```
