# Level Design - 关卡设计技巧

> **用途**: 在无限流、游戏文、副本流、快穿文中，设计有趣、平衡且致命的关卡  
> **适用流派**: 无限流、网游、惊悚乐园、快穿系统  
> **关键特征**: 规则限制、地形利用、隐藏要素、Boss机制

---

## 一、核心特征分析

### 1.1 定义

关卡设计（Level Design），是把剧情发生的舞台"游戏化"。它不仅仅是一个背景板，而是一个充满**规则、陷阱、奖励和挑战**的互动空间。好的关卡本身就是主角的敌人，也是主角的游乐场。

**核心价值**:
- **新鲜感**: 每个副本都是一个新世界（末日、古堡、太空），避免审美疲劳
- **策略性**: 限制主角的能力（禁魔/禁科技），逼迫主角动脑子
- **节奏感**: 探索 -> 战斗 -> 解谜 -> Boss战 -> 结算，天然的爽文节奏
- **团队配合**: 不同的地形和机制需要不同的职业搭配

### 1.2 类型划分

**按目标**:
1. **生存类**: 活过7天，或者逃离某个区域（《生化危机》模式）。
2. **解谜类**: 找出凶手，或者解开机关（密室逃脱模式）。
3. **对抗类**: 多方阵营大逃杀，吃鸡模式（《饥饿游戏》模式）。
4. **守卫类**: 保护某个NPC或据点（塔防模式）。

**按结构**:
1. **箱庭式**: 小而精的封闭空间，细节丰富（古堡、列车）。
2. **线性**: 一条路走到黑，过五关斩六将（闯塔）。
3. **开放世界**: 自由探索，触发支线（沙盒）。

### 1.3 常见误区

❌ **地图太空**: 走了半天什么都没发生，纯粹水字数  
❌ **缺乏指引**: 读者和主角都不知道该干嘛，像无头苍蝇  
❌ **难度失衡**: 小怪秒杀主角（劝退），BOSS被主角秒杀（烂尾）  
❌ **规则无效**: 定了规则（如不能说话），结果主角说话了没事，规则成了摆设

## 二、Top作品解析

### 2.1 经典案例1: 《惊悚乐园》

**作者**: 三天两觉  
**核心手法**: 剧本杀+隐藏世界观

**经典关卡（霹雳偶戏馆）**:
> 表面：一个打败布袋戏人偶的副本。
> 机制：由于主角是"数据视角"，他发现了人偶背后的操纵线。
> **隐藏剧情**: 这是一个被封印的神魔战场。人偶只是封印物。
> 主角通过破坏场景中的隐藏道具，触发了真·结局，获得了隐藏装备。

**分析**:
- **表里世界**: 表面一套任务，背地里一套真相
- **探索奖励**: 为"舔图党"（探索角落）准备丰厚奖励
- **文风适配**: 关卡风格与原著（霹雳布袋戏）完美融合
- **可复用要素**: "隐藏任务大于主线任务"

**为什么有效**:
1. 满足了读者的探索欲
2. 证明了主角比系统更聪明

### 2.2 经典案例2: 《全职高手》

**作者**: 蝴蝶蓝  
**核心手法**: Boss机制+环境利用

**经典关卡（烈焰森林BOSS战）**:
> BOSS是一个红带女巫。
> 机制：会召唤陨石雨，地形全是着火的树。
> 打法：叶修把小怪引到BOSS的陨石雨里砸死（借刀杀人）。
> 利用地形的高低差，卡住BOSS的视角（卡怪）。

**分析**:
- **环境互动**: 地形不是死的，是可以被利用的武器
- **仇恨控制**: OT（仇恨失控）、连击、硬直，游戏术语的文学化应用
- **战术**: 只要机制利用得好，低级也能打高级
- **可复用要素**: "不是靠数值，是靠操作和脑子"

**为什么有效**:
1. 极强的专业感（游戏理解）
2. 充满了"脏套路"，看起来很爽

### 2.3 经典案例3: 《无限恐怖》

**作者**: zhttty  
**核心手法**: 团战+非对称信息

**经典关卡（咒怨）**:
> 背景：鬼怪片，必死局。
> 机制：不能离开这栋房子太远，否则抹杀。但鬼就在房子里。
> 变量：引入了另一队轮回者（恶魔队）。
> 冲突：既要防着鬼，又要防着人。还需要保护新人（累赘）。

**分析**:
- **多重限制**: 空间限制 + 时间限制 + NPC保护
- **博弈**: 两个团队的信息是不对等的
- **分支**: 拿到"佛经"可以克制鬼，但这需要去危险的寺庙
- **可复用要素**: "多方混战模型"

**为什么有效**:
1. 这种复杂的局势能最大程度激发人性
2. 绝境求生

## 三、实战技法

### 3.1 基础公式

**Boss机制设计公式**:
```
BOSS特性（高攻低防/无敌护盾） + 场景弱点（利用陷阱/打破水晶） + 阶段变化（狂暴模式） = 合格的战斗

示例：
BOSS全身无敌。
场景周围有四个柱子。
引诱BOSS撞柱子，眩晕5秒，破防。
重复3次，BOSS进入狂暴，柱子全碎。
硬拼。
```

**探索收益公式**:
```
高风险区域（怪物巢穴） + 必须的钥匙/强力道具 + 守护怪 = 支线探索

示例：
主线门锁了。
钥匙在地下室。
地下室全是丧尸。
主角去地下室拿到钥匙，顺便捡到一把喷子（奖励）。
```

### 3.2 进阶技巧

#### 1. 动态难度 (Dynamic Difficulty)

系统看主角太顺了，给他加点料。
- "检测到玩家实力过强，触发隐藏难度：地狱模式。"
- 奖励翻倍，但怪物全部变异。
- **爽点**: 官方认证的强者/富贵险中求。

#### 2. 规则破坏者

关卡规定"禁止飞行"。
主角没有飞，主角把地心引力改了。
主角卡了系统的Bug，跳过了整个关卡直接见BOSS。
**爽点**: 智商碾压系统设计者。

#### 3. 存档点欺诈

以为到了安全屋（存档点）。
结果安全屋里藏着最大的BOSS。
打破玩家的安全感。

### 3.3 常见套路

#### 套路1: 怪物内讧

关卡里有两波怪物（兽人和亡灵）。
它们互相对立。
主角稍微挑拨一下，坐山观虎斗。
最后出来收人头。

#### 套路2: 限制解除

前90%的时间，主角被限制了能力（比如变成了凡人）。
极度压抑。
最后关头，限制解除（拿回了装备/恢复了法力）。
瞬间秒杀全场。
**爽点**: 欲扬先抑的极致。

## 四、写作要点

### 节奏控制

**进场**: 介绍环境，渲染氛围，交代主线。  
**遇敌**: 小怪热身，熟悉机制。  
**探索**: 发现隐藏要素，支线剧情。  
**决战**: 机制全开，底牌尽出。  
**结算**: 点算收获（舔包），复盘。

### 情绪设计

**好奇**: "这里面藏着什么？"  
**贪婪**: "那个宝箱看起来好诱人（虽然有陷阱）。"  
**成就感**: "这么难的机关都被我破解了。"

### 陷阱规避

❌ **避免**: 迷宫写得太复杂，读者没地图看不懂空间关系  
✅ **改为**: 简化空间结构，或者用标志性参照物

❌ **避免**: 主角一路平推，毫无波澜  
✅ **改为**: 即使是无敌流，也要遇到一些机制上的麻烦（如保护人质）

❌ **避免**: 关卡设计与剧情割裂  
✅ **改为**: 关卡本身就在讲故事（通过尸体、日记碎片）

## 五、检查清单

写完后自检：

- [ ] 关卡是否有明确的主线目标（逃生/击杀）？
- [ ] 是否利用了环境地形（高低差/水火）？
- [ ] BOSS是否有特殊的战斗机制（不只是血厚）？
- [ ] 是否安排了隐藏要素/支线奖励？
- [ ] 是否限制了主角的某些能力（迫使变通）？
- [ ] 结尾是否有丰厚的结算奖励？
