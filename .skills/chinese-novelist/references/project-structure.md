## 📁 标准文件结构 (8 件套)

| 序号 | 文件名 | 作用 | 核心价值 | 模板来源 |
|:-----|:-------|:-----|:---------|:---------|
| 00 | `00-策划书.md` | 确立一句话梗概、目标受众、对标作品 | **市场定位**：确保写出来的东西有人看 | `templates/pitch-template.md` |
| 01 | `01-世界观圣经.md` | 定义力量体系、经济系统、势力博弈 | **底层逻辑**：防止战力崩坏和设定冲突 | `templates/world-bible-template.md` |
| 02 | `02-人物深度档案.md` | 引入"幽灵"、"谎言"、"想要vs需要" | **人物弧光**：确保主角有灵魂，有成长 | `templates/character-depth-template.md` |
| 03 | `03-分章细纲.md` | 基于 15节拍表 (Save the Cat) 规划 | **结构节奏**：确保故事高潮迭起，不记流水账 | `templates/beat-sheet-template.md` |
| 04 | `04-伏笔追踪表.md` | 记录活跃悬念和已埋设的伏笔 | **逻辑严密**：防止挖坑不填 | `templates/clues-tracker-template.md` |
| 05 | `05-时间线.md` | 管理绝对时间和相对时间 | **时空连贯**：避免时间穿越bug | `templates/timeline-template.md` |
| 06 | `06-细节追踪表.md` | 跟踪人物/物品/地点/关系/能力等一致性状态 | **一致性引擎**：防止长文矛盾与穿帮 | `templates/detail-tracker-template.md` |
| 07 | `07-项目状态.md` | 会话间状态同步、当前进度 | **上下文连续**：跨会话不丢失进度 | `templates/project-status-template.md` |

> 可选补充文件：`06-素材碎片.md`（灵感池/金句库），不作为运行时必需状态源。

## 🔄 渐进式披露协议 (Progressive Disclosure Protocol)

### 默认行为
AI 在初始化项目时，**必须**遵循以下步骤：

1.  **每次只生成 1 个文件**。
2.  **生成后，向用户确认**：
    ```
    ✅ [序号/8] `00-策划书.md` 已生成。
    👉 是否继续生成 `01-世界观圣经.md`？(Y/跳过/全部生成)
    ```
3.  **用户回复 "Y" 或 "继续"**：进入下一个文件。
4.  **用户回复 "跳过"**：跳过当前文件，进入下一个。
5.  **用户回复 "全部生成"**：批量完成剩余所有文件。

### 快捷模式
- 用户说 **"快速启动"** 或 **"全部生成"**：跳过确认，一次性生成全部 8 个文件。
- 用户说 **"只要策划书"**：仅生成 `00-策划书.md`。

## 🎭 两种模式定义 (Mode Definitions)

### 引导式 (Guided Mode)
**触发词**：`我是新手` / `带我入门` / `引导模式` / `开新书`

**行为**：
1.  AI 扮演"文学顾问"，通过 **5 轮对话** 收集信息：
    - Q1: 核心概念 (一句话梗概)
    - Q2: 流派与风格
    - Q3: 主角设定 (名字/目标/缺陷)
    - Q4: 世界观 (力量体系/代价)
    - Q5: 确认并开始生成
2.  收集完成后，**渐进式生成** 8 个文件。
3.  **每生成 1 个文件后确认**。

**检查清单** (`modes/guided-mode-checklist.md`)：
```markdown
- [ ] 已询问核心概念？
- [ ] 已询问流派风格？
- [ ] 已询问主角设定？
- [ ] 已询问世界观？
- [ ] 已汇总并请求用户确认？
→ 全部完成后，开始渐进式文件生成。
```

---

### 专家式 (Expert Mode)
**触发词**：`专家模式` / `创建项目：[书名]` / `空项目`

**行为**：
1.  **不进行任何访谈**。
2.  **一次性生成** 8 个空文件框架（仅标题和占位符）。
3.  用户自行填写内容。

**输出示例**：
```
✅ 专家模式：已创建 novels/[书名]/ 项目框架。
生成文件：00-策划书.md ~ 07-项目状态.md (共8个)
所有文件均为空模板，请自行填写。
```

## 📋 文件生成顺序 (Generation Order)

```mermaid
graph LR
    A[00-策划书] --> B[01-世界观]
    B --> C[02-人物档案]
    C --> D[03-分章细纲]
    D --> E[04-伏笔追踪]
    E --> F[05-时间线]
    F --> G[06-素材碎片]
    G --> H[07-项目状态]
```

**依赖关系**：
- `00-策划书` 是所有文件的基础（流派、受众）。
- `01-世界观` 依赖策划书中的世界类型。
- `02-人物档案` 依赖世界观中的力量体系。
- `03-分章细纲` 依赖人物和世界观。
- `04-07` 为辅助文件，可并行生成。

## 🔗 Hook 调用规范 (Hook References)

当 AI 需要生成某个文件时，**必须**先读取对应模板：

```
生成 00-策划书.md 前：
→ Read("references/templates/pitch-template.md")

生成 01-世界观圣经.md 前：
→ Read("references/templates/world-bible-template.md")

... (以此类推)
```

## ✅ 验证规则 (Validation Rules)

`validate_integrity.py` 必须检查：
1.  此文件中的 8 个模板路径是否全部存在。
2.  `quick_start.py` 是否生成全部 8 个文件。
3.  `SKILL.md` 中的 Hook 引用是否与此文件一致。
